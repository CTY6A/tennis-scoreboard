## Плюсы

- Имена классов, методов и переменных понятны и отражают их назначение
- Необычное распределение классов по пакетам
- В целом хорошее разделение на слои (Servlet -> Service -> DAO)
- Использование criteria в репозитории
- Использование фильтра для обработки исключений
- Использование DTO
- Есть валидация имён игроков
- Логика проверки имён вынесена в отдельный валидатор
- Есть мапперы
- Используется MapStruct
- Используется Lombok для уменьшения boilerplate-кода
- Работает фильтрация матчей по имени игрока
- Работает пагинация на странице поиска матчей (хотя и стоит её доработать)
- Страницы JSP лежат внутри `/WEB-INF`
- База данных наполняется тестовыми данными при старте приложения
- Наличие тестов
- Версия java вынесена в секцию properties в pom.xml
- Деплой приложения
